# Техническая документация: F3.2 - Опциональные поля формы

**Версия:** 1.0  
**Компонент:** `BookingForm` (расширение)  
**Файл:** `booking-form.jsx`  

## Опциональные поля

### 1. Telegram
- **Имя:** telegram
- **Валидация:** Должен начинаться с @ (если заполнен)
- **Placeholder:** @username
- **Подсказка:** "Для уведомлений о встрече в Telegram"

### 2. Дополнительные участники
- **Имя:** additionalParticipants
- **Тип:** textarea (3 строки)
- **Валидация:** Каждый email должен быть корректным
- **Формат:** через запятую или с новой строки
- **Placeholder:** colleague@example.com, manager@example.com
- **Подсказка:** "Email коллег, которые присоединятся к встрече"

### 3. Вопросы для обсуждения
- **Имя:** questions
- **Тип:** textarea (4 строки)
- **Валидация:** Максимум 500 символов
- **Счётчик:** Показывает количество символов (X / 500)
- **Цвет счётчика:** Красный при > 450 символов
- **Placeholder:** "Укажите вопросы, которые хотите обсудить..."
- **Подсказка:** "Это поможет нам подготовиться к встрече"

## Поведение

### Валидация опциональных полей

#### Telegram
```javascript
if (value.trim() && !value.trim().startsWith('@')) {
  return 'Telegram должен начинаться с @';
}
```

#### Дополнительные участники
```javascript
const emails = value.split(/[,\n]/).map(e => e.trim()).filter(e => e);
for (const email of emails) {
  if (!validateEmail(email)) {
    return `Некорректный email: ${email}`;
  }
}
```

#### Вопросы
```javascript
if (value.length > 500) {
  return 'Максимум 500 символов';
}
```

### Отправка формы

Опциональные поля **НЕ блокируют** отправку формы.
Форма может быть отправлена с пустыми опциональными полями.

## State

```javascript
formData: {
  // Обязательные
  firstName: '',
  lastName: '',
  email: '',
  phone: '',
  
  // Опциональные
  telegram: '',
  additionalParticipants: '',
  questions: ''
}
```

## Визуальные отличия

- Метка "(необязательно)" рядом с названием поля
- Серый цвет для текста "(необязательно)"
- Подсказки под каждым полем
- Счётчик символов для вопросов

## Интеграция

Опциональные поля автоматически включены в компонент `BookingForm`.
Никаких дополнительных props не требуется.

```jsx
<BookingForm onSubmit={(data) => {
  console.log(data.telegram); // может быть пустым
  console.log(data.additionalParticipants); // может быть пустым
  console.log(data.questions); // может быть пустым
}} />
```
