# Список функций системы бронирования встреч

**Проект:** Система онлайн бронирования демо Scroll Tool  
**Версия:** 1.0  
**Дата:** 14 апреля 2026

## 🎯 MVP (Минимально жизнеспособный продукт)

### БЛОК 1: Просмотр календаря и свободных слотов

#### F1.1: Отображение календаря
**Описание:** Календарь для выбора даты встречи

**Требования:**
- Отображение текущего месяца
- Переключение между месяцами (вперед/назад)
- Выделение текущей даты
- Блокировка прошедших дат
- Отображение только рабочих дней (Пн-Пт)
- Выходные (Сб-Вс) не показываются

**Технические детали:**
- Компонент: React Calendar
- Валидация: дата >= сегодня
- Фильтр: только Пн-Пт

**Приоритет:** 🔴 Критический (без этого нет системы)

---

#### F1.2: Проверка доступности слотов
**Описание:** Система определяет свободные временные слоты

**Требования:**
- Рабочее время: 10:00 - 19:00 МСК
- Интервал слотов: 30 минут (10:00, 10:30, 11:00...)
- Проверка 4 календарей Mail.ru одновременно:
  - Владелец (v.pinigin@scroll-tool.ru)
  - Методист 1
  - Методист 2
  - Менеджер
- Слот свободен = минимум 1 человек из команды свободен
- Учет выбранной длительности (30/45/60 мин)
- **Минимальная дата для записи: завтра** (сегодняшний день всегда заблокирован)

**Логика доступности:**
```
Слот свободен ЕСЛИ:
  (Методист 1 свободен ИЛИ Методист 2 свободен ИЛИ Владелец свободен)
  И
  Менеджер не имеет слишком много параллельных встреч
```

**Технические детали:**
- API: GET /api/availability?date=2026-04-15&duration=60
- Интеграция: CalDAV для Mail.ru
- Кэширование: 5 минут

**Зависимости:**
- Требует: CalDAV интеграция (F6.1)

**Приоритет:** 🔴 Критический

### БЛОК 3: Форма бронирования

#### F3.1: Обязательные поля клиента
**Описание:** Сбор основной информации

**Поля:**
1. **Имя** (обязательно)
   - Валидация: не пустое, мин 2 символа
   
2. **Фамилия** (обязательно)
   - Валидация: не пустое, мин 2 символа
   
3. **Email** (обязательно)
   - Валидация: формат email (regex)
   - Пример: ivan@example.com
   
4. **Телефон** (обязательно)
   - Валидация: российский формат
   - Маска ввода: +7 (___) ___-__-__
   - Пример: +7 (999) 123-45-67

**UI/UX:**
- Автофокус на первое поле
- Inline валидация (при потере фокуса)
- Красные подсветки ошибок
- Иконки статуса (✓ или ✗)

**Приоритет:** 🔴 Критический

### БЛОК 4: Распределение встречи по команде

#### F4.1: Алгоритм round-robin для методистов
**Описание:** Чередование методистов для равномерной нагрузки

**Логика:**
```
1. Получить last_assigned из таблицы team_rotation
2. Если last_assigned = 'methodist1':
   - Проверить свободен ли Методист 2
   - Если да → назначить Методиста 2
   - Если нет → проверить Методиста 1
3. Если last_assigned = 'methodist2':
   - Проверить свободен ли Методист 1
   - Если да → назначить Методиста 1
   - Если нет → проверить Методиста 2
4. Обновить last_assigned в таблице
```

**Технические детали:**
- Таблица БД: team_rotation
- Поле: last_assigned ('methodist1' или 'methodist2')
- Атомарная операция (transaction)

**Приоритет:** 🔴 Критический

---

#### F4.2: Fallback на владельца
**Описание:** Назначение владельца если оба методиста заняты

**Логика:**
```
ЕСЛИ Методист 1 занят И Методист 2 занят:
  Проверить владельца
  ЕСЛИ владелец свободен:
    → назначить владельца
  ИНАЧЕ:
    → слот недоступен (этого не должно произойти т.к. фильтр F1.2)
```

**Приоритет:** 🟡 Высокий

---

#### F4.3: Автоматическое добавление менеджера
**Описание:** Менеджер всегда добавляется во встречу

**Логика:**
- Независимо от назначенного ведущего
- Менеджер добавляется в календарное событие
- НЕ проверяем занятость менеджера (он может быть на параллельных встречах)

**Приоритет:** 🟡 Высокий

### БЛОК 5: Создание встречи

#### F5.1: Сохранение в базу данных
**Описание:** Запись бронирования в Supabase

**Данные для сохранения:**
- Дата и время встречи
- Длительность (30/45/60)
- Данные клиента (имя, фамилия, email, телефон, telegram)
- Дополнительные участники (массив)
- Вопросы для обсуждения
- Назначенный ведущий (email + role)
- Email менеджера
- ID встречи МТС-Линк
- URL встречи МТС-Линк
- ID событий в календарях
- Статус (confirmed)
- Timestamps

**Технические детали:**
- Таблица: bookings
- UUID для id
- Timestamps: created_at, updated_at

**Приоритет:** 🔴 Критический

---

#### F5.2: Создание встречи в МТС-Линк
**Описание:** Генерация видеоконференции

**Требования:**
- Название: "Демо Scroll Tool - [Имя Фамилия]"
- Дата и время
- Длительность (выбранная)
- Участники:
  - Назначенный ведущий
  - Менеджер
  - Клиент (email)
  - Дополнительные участники клиента

**API запрос:**
```javascript
POST /api/mts-link/meetings
{
  title: "Демо Scroll Tool - Иван Петров",
  start_time: "2026-04-15T14:00:00+03:00",
  duration: 60,
  participants: [
    { email: "methodist1@scroll-tool.ru", role: "host" },
    { email: "manager@scroll-tool.ru", role: "participant" },
    { email: "ivan@example.com", role: "participant" }
  ]
}
```

**Результат:**
- meeting_id
- join_url (ссылка для подключения)

**Зависимости:**
- МТС-Линк API ключ

**Приоритет:** 🔴 Критический

### БЛОК 6: Синхронизация с календарями

#### F6.1: Подключение к Mail.ru CalDAV
**Описание:** Интеграция с 4 календарями через CalDAV

**Календари:**
1. v.pinigin@scroll-tool.ru
2. methodist1@... (TBD)
3. methodist2@... (TBD)
4. manager@... (TBD)

**Функции:**
- Чтение событий (для проверки занятости)
- Создание событий
- Обновление событий

**Технические детали:**
- Библиотека: tsdav (npm)
- Протокол: CalDAV
- URL: https://caldav.mail.ru
- Auth: Basic (email + пароль приложения)

**Credentials:**
```env
MAILRU_OWNER_EMAIL=v.pinigin@scroll-tool.ru
MAILRU_OWNER_PASSWORD=пароль_приложения
# ... аналогично для остальных
```

**Зависимости:**
- Пароли приложений от всех 4 аккаунтов

**Приоритет:** 🔴 Критический

---

- Дата и время
- Длительность
- Участники: все email участников
- Напоминания: за 1 час, за 15 минут

**Технические детали:**

### БЛОК 7: Уведомления

#### F7.1: Email клиенту (календарное приглашение)
**Описание:** Отправка .ics файла на email

**Содержание письма:**
- Тема: "✅ Встреча подтверждена: Демо Scroll Tool"
- HTML шаблон:
  - Приветствие
  - Подтверждение бронирования
  - Детали встречи (дата, время, длительность)
  - Имя ведущего (без email)
  - Ссылка МТС-Линк (кнопка)
  - Контакты для связи
- Вложение: .ics файл (календарное приглашение)

**Календарное приглашение (.ics):**
- Клиент может добавить в свой календарь (Google/Outlook/Mail.ru)
- Автоматически создается событие
- Напоминания встроены

**Технические детали:**
- Сервис: SendGrid или Mail.ru SMTP
- Библиотека: ics (для генерации .ics)
- Формат: HTML email + .ics attachment

**Зависимости:**
- SendGrid API ключ ИЛИ Mail.ru SMTP credentials

**Приоритет:** 🔴 Критический
