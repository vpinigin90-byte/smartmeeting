**Версия:** 1.0  
**Дата:** 14 апреля 2026  
**Компонент:** `DurationSelector`  
**Файл:** `duration-selector.jsx`  

---

## 1. Описание функции

### 1.1 Назначение
Компонент для выбора длительности демонстрационной встречи с тремя вариантами:
- 30 минут (экспресс-демо)
- 45 минут (стандартное демо)
- 60 минут (полное демо с вопросами) - **по умолчанию**

### 1.2 Функциональные требования
- ✅ Отображение 3 вариантов длительности
- ✅ Визуальное выделение выбранного варианта
- ✅ 60 минут выбрано по умолчанию
- ✅ Бейдж "РЕКОМЕНДУЕМ" на варианте 60 минут
- ✅ Callback при изменении длительности
- ✅ Кнопка "Продолжить к выбору даты"
- ✅ Отображение текущего выбора

## 4. Основные функции

### 4.1 `handleDurationSelect(duration)`

**Назначение:** Обработчик выбора длительности

**Параметры:**
- `duration` (number) - выбранная длительность (30, 45 или 60)

**Возвращает:** 
- `void`

**Алгоритм:**
```javascript
const handleDurationSelect = (duration) => {
  // 1. Обновляем локальный state
  setSelectedDuration(duration);
  
  // 2. Вызываем callback если передан
  if (onDurationChange) {
    onDurationChange(duration);
  }
};
```

**Пример:**
```javascript
handleDurationSelect(45);
// → selectedDuration = 45
// → onDurationChange(45) вызван
```

## 7. Интеграция в приложение

### 7.1 Standalone использование

```jsx
import DurationSelector from './duration-selector';

function App() {
  return <DurationSelector />;
}
```

### 7.2 С callback обработчиком

```jsx
import DurationSelector from './duration-selector';

function App() {
  const handleDurationChange = (duration) => {
    console.log(`Клиент выбрал: ${duration} минут`);
    // Сохранить в state, отправить в API, и т.д.
  };

  return (
    <DurationSelector onDurationChange={handleDurationChange} />
  );
}
```

### 7.3 Интеграция с другими компонентами

```jsx
import { useState } from 'react';
import DurationSelector from './duration-selector';
import BookingCalendar from './calendar-component';

function BookingFlow() {
  const [step, setStep] = useState('duration'); // 'duration' или 'calendar'
  const [selectedDuration, setSelectedDuration] = useState(60);

  return (
    <>
      {step === 'duration' && (
        <DurationSelector 
          onDurationChange={(duration) => {
            setSelectedDuration(duration);
            setStep('calendar');
          }}
        />
      )}
      
      {step === 'calendar' && (
        <BookingCalendar duration={selectedDuration} />
      )}
    </>
  );
}
```

## 8. UX детали

### 8.1 Бейдж "РЕКОМЕНДУЕМ"

- Отображается только на варианте 60 минут
- Позиционирование: над кнопкой, слева
- Цвет: градиент фиолетовый
- Цель: направить пользователя к оптимальному выбору

### 8.2 Радио-кнопка (custom)

- Круглая иконка справа вверху кнопки
- Пустой круг = не выбрано
- Синий круг с белой точкой = выбрано
- Анимация при переключении

### 8.3 Информационный блок

- Показывает текущий выбор: "Вы выбрали: 60 минут"
- Дополнительная информация: "Встреча пройдет онлайн через МТС-Линк"

## 12. Известные ограничения

### 12.1 Текущие ограничения

1. **Жестко заданные варианты**
   - Только 3 варианта: 30/45/60 минут
   - Нельзя добавить другие значения без изменения кода
   - **Решение (Phase 2):** Принимать массив вариантов через props

2. **Язык интерфейса**
   - Только русский язык
   - **Решение:** i18n (react-i18next)

3. **Нет валидации**
   - Нет проверки корректности выбранного значения
   - **Решение:** Добавить PropTypes или TypeScript
